1- ¿Cuáles son las principales funciones de un kernel en la arquitectura de un sistema operativo y cómo interactúa con el hardware

.El kernel es el núcleo del sistema operativo, controlando directamente el hardware y proporcionando servicios esenciales. Sus funciones clave son: Gestión de Procesos: Crea, planifica y gestiona la ejecución de programas. Gestión de Memoria: Asigna y controla el uso de la memoria principal por parte de los programas. Gestión de Dispositivos: Controla el acceso y la operación del hardware a través de controladores. Gestión del Sistema de Archivos: Organiza y administra el acceso a los datos en almacenamiento. Gestión de Recursos: Distribuye los recursos del sistema (CPU, memoria, dispositivos) entre los programas.

2- ¿cómo se gestionan los procesos y cuál es la importancia de la multitarea en la eficiencia del sistema?

La gestión de procesos implica la creación, planificación, sincronización y terminación de las instancias de programas en ejecución (procesos). El sistema operativo mantiene información sobre cada proceso en un Bloque de Control de Proceso (PCB) y utiliza un planificador para decidir qué proceso usa la CPU y cuándo, realizando cambios de contexto rápidos entre ellos. La multitarea es crucial porque permite al sistema operativo ejecutar múltiples procesos aparentemente al mismo tiempo, alternando rápidamente entre ellos. Esto mejora la eficiencia del sistema al: Maximizar el uso de la CPU: Evita que la CPU esté inactiva esperando operaciones de E/S al ejecutar otro proceso. Aumentar la productividad del usuario: Permite ejecutar varias aplicaciones simultáneamente. Mejorar la responsividad: Mantiene el sistema interactivo incluso cuando hay tareas pesadas en ejecución. Facilitar la compartición de recursos: Permite que múltiples procesos compartan los recursos del sistema de manera efectiva..

Qué medidas de seguridad se pueden implementar a nivel de sistema operativo para proteger una red, especialmente en entornos empresariales

Firewall del Sistema Operativo: Filtra el tráfico de red entrante y saliente en el propio dispositivo, bloqueando conexiones no deseadas. Gestión de Usuarios y Permisos: Controla quién accede al sistema y qué acciones puede realizar cada usuario, basándose en privilegios mínimos. Autenticación Fuerte (Contraseñas y MFA): Requiere credenciales robustas y, idealmente, pasos de verificación adicionales para confirmar la identidad del usuario. Actualizaciones y Parches Regulares: Mantiene el sistema operativo y el software al día para corregir vulnerabilidades conocidas
